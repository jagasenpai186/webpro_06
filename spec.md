# 開発者向け仕様書（仮）

##　データ構造

| 属性名         | 型      | 説明            |
| ----------- | ------ | ------------- |
| id          | Number | データを一意に識別するID |
| name        | String | 一覧画面に表示する名称   |
| description | String | 詳細画面に表示する説明   |
| category    | String | データの分類        |

## ページ遷移
ページ遷移方法

一覧画面に表示された名称をクリックすることで詳細ページへ遷移する．
ページ内リンクによる画面遷移を行う．
| 画面   | URL              | 内容            |
| ---- | ---------------- | ------------- |
| 一覧画面 | /items           | データ一覧を表示      |
| 詳細画面 | /items/{id}      | 選択したデータの詳細を表示 |
| 追加画面 | /items/new       | 新規データの入力      |
| 編集画面 | /items/{id}/edit | データの編集        |

HTTPメソッドとリソースについては以下の通りである．
| 機能    | HTTPメソッド | リソース            |
| ----- | -------- | --------------- |
| 一覧取得  | GET      | /api/items      |
| 詳細取得  | GET      | /api/items/{id} |
| データ追加 | POST     | /api/items      |
| データ編集 | PUT      | /api/items/{id} |
| データ削除 | DELETE   | /api/items/{id} |
## 編集機能
追加などの編集については以下の通りである．
| 操作 | 表示画面             |
| -- | ---------------- |
| 追加 | 一覧画面（追加済みデータを含む） |
| 編集 | 詳細画面（更新後の内容）     |
| 削除 | 一覧画面（削除後の一覧）     |

## リソースごとの機能詳細
- /api/items

`GET`
登録されている全データを取得する．

`POST`
新しいデータを追加する．
id はサーバ側で自動的に付与する．

- /api/items/{id}

`GET`
指定されたIDのデータを取得する．

`PUT`
指定されたIDのデータを編集する．

`DELETE`
指定されたIDのデータを削除する．
